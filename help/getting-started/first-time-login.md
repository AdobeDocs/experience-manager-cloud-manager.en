# First time login {#first-time-login}

After you have completed the general configuration for [!UICONTROL Cloud Manager], you are ready to use it.

>[!NOTE]
>
>Depending on the roles assigned in [!UICONTROL Cloud Manager] and the state of the application, the screens you see vary from those in the illustrations.

## Log in for the first time {#logging-in-for-the-first-time}

1. Sign into Cloud Manager at [experience.adobe.com](https://experience.adobe.com/experiencemanager). To learn more, see [About Adobe Experience Hub](https://experienceleague.adobe.com/en/docs/experience-manager-65/content/experience-hub/experience-hub)
1. In the left side panel, click **Cloud Manager**.

   ![Cloud Manager in left side panel in Experience Manager](/help/getting-started/assets/cloud-manager-experiencemanager.png)

1. If your program is not yet configured, click **Setup Program** in the [call-to-action](/help/getting-started/navigation.md#cta) to configure your program description and define key performance indicators (KPIs). If your program is already configured, the CTA offers additional next steps.

## Cloud Manager activity {#activity}

Cloud Manager provides a consolidated view of a program's activity. It lists all CI/CD pipeline executions, both production and non-production. This functionality provides information about historical and current activity.

>[!TIP]
>
>For details on how to use the Cloud Manager UI, see [Navigating the Cloud Manager UI](/help/getting-started/navigation.md).

>[!NOTE]
>
>By default, a user with access to an AEM environment also has the Cloud Manager User role. This role does not grant the user access to the program details view. A user with only the Cloud Manager User role can navigate using the program menu options to the AEM environment author URL (if environments exist). These users must contact their administrators if they want to get program-level access.
